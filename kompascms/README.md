Кастомная админ-CRM для kompasmigracji.com: дашборд с аналитикой, реестр
участников профсоюза, личный кабинет каждого участника, лиды воронки,
управление контентом сайта, команда CMS. Встроена в существующий `/admin`,
ничего не ломает.

---

## Архитектура — путь архитектора

KompasCRM спроектирована как **изолированный модуль**, чтобы безопасно встать
в живой проект:

- Все таблицы — с префиксом `kompas_`. Не конфликтуют с твоими таблицами.
- Все API — под `/api/admin/*`. Не пересекаются с твоими роутами.
- Своя авторизация — JWT в httpOnly-cookie, отдельная от авторизации сайта.
- Стили — с префиксом `kc-` в одном файле. Не текут в стили сайта.

**Роли:** `admin` (всё) · `moderator` (участники, лиды) · `member` — участник
профсоюза, видит только свой кабинет `/admin/me`.

```
db/schema.sql               таблицы Postgres (kompas_*)
db/seed.sql                 блоки контента
scripts/create-admin.mjs    создание первого админа
lib/db.js                   пул Postgres
lib/auth.js                 JWT, bcrypt, requireAuth
lib/rbac.js                 роли, права, навигация
middleware.js               защита /admin/*
app/api/admin/
  auth/login                вход
  auth/logout               выход
  auth/me                   текущий пользователь
  auth/password             смена своего пароля
  stats                     агрегаты дашборда
  members  + members/[id]   реестр и карточки участников
  leads                     лиды воронки
  content                   блоки контента сайта
  team                      администраторы и модераторы
app/admin/login             страница входа
app/admin/(panel)/
  page.jsx                  дашборд с аналитикой
  members, members/[id]     участники профсоюза
  leads                     лиды
  content                   контент сайта
  me                        кабинет участника
  settings                  команда, смена пароля
components/admin/           каркас (Shell) и UI-компоненты
styles/kompascrm.css        тема CMS
```

## Установка

### 1. Зависимости

```bash
npm install pg jose bcryptjs
```

### 2. Скопировать файлы в проект

Перенеси папки `lib/`, `components/`, `app/`, `db/`, `scripts/`, `styles/` и
файл `middleware.js` в свой проект kompasmigracji.com.

- Если проект использует каталог `src/` — клади внутрь `src/`.
- Алиас `@/` должен указывать на корень кода (`jsconfig.json` →
  `"paths": { "@/*": ["./*"] }` или `["./src/*"]`). В проектах от
  `create-next-app` это уже настроено.
- `middleware.js` кладётся рядом с `app/` (или в `src/`).

### 3. Переменные окружения

Скопируй `.env.example` → `.env.local`, заполни `DATABASE_URL` и `JWT_SECRET`.

### 4. База данных

```bash
psql "$DATABASE_URL" -f db/schema.sql
psql "$DATABASE_URL" -f db/seed.sql
node scripts/create-admin.mjs admin@kompasmigracji.com ТвойПароль "Александр"
```

### 5. Запуск

```bash
npm run dev
```

Открой `/admin/login` — войди под созданным админом.

## Твои старые наработки в /admin

Они **сохраняются**. Чтобы старые страницы получили оформление новой CMS —
перенеси их папки внутрь `app/admin/(panel)/`. Скобки `(panel)` — это route
group: URL не меняется (`/admin/что-то` остаётся `/admin/что-то`), но страница
автоматически попадает в каркас CMS с сайдбаром и топбаром.

Старые страницы, которые НЕ нужно оборачивать в CMS — оставь в `app/admin/`
вне `(panel)`, они продолжат работать как раньше.

## Что работает

**Полностью рабочие модули:**

- Авторизация с ролями, защита всех маршрутов `/admin/*`, журнал действий.
- Дашборд: метрики участников/лидов/дел/взносов, динамика регистраций за
  14 дней, воронка лидов по источникам (бот / сайт / FB / IG), собираемость
  взносов.
- Участники профсоюза: реестр с поиском, создание (с временным паролем),
  карточка, редактирование, удаление.
- Кабинет участника `/admin/me`: профиль, взносы, обращения; участник сам
  редактирует свои контакты (телефон, город).
- Лиды: список с фильтром по статусу, смена статуса.
- Контент сайта `/admin/content`: четыре блока юридического чек-листа
  (оферта, цены, регламент, политика приватности) — редактирование,
  сохранение, публикация / снятие с публикации.
- Команда CMS `/admin/settings`: список администраторов и модераторов,
  приглашение нового члена команды с временным паролем, смена своего пароля.

**Дорожная карта:** refresh-токены и отзыв сессий, модуль юридических дел
`/admin/cases`, экспорт реестра участников в CSV, оплата взносов через
Przelewy24, двухфакторная аутентификация.

## Интеграционные точки (где нужна твоя реальность)

1. **Лиды.** Модуль читает таблицу `kompas_leads`. Чтобы сюда попадали лиды
   бота/сайта/FB/IG — добавь в свой `/api/leads` запись и в `kompas_leads`,
   либо в `app/api/admin/leads/route.js` поменяй запрос на свою таблицу лидов.
2. **БД-слой.** Если у тебя уже есть подключение к Postgres — замени
   `lib/db.js` на свой модуль (нужна функция `q(sql, params)` и `one(...)`).
3. **Контент на сайте.** Модуль `/admin/content` сохраняет блоки в
   `kompas_content`. Чтобы они отобразились на публичных страницах сайта —
   читай эту таблицу по `slug` там, где выводишь оферту/цены/регламент.

## Честно об ограничениях

Это рабочая CMS, собранная без доступа к твоему репозиторию — поэтому она
изолирована и встаёт чисто, но интеграционные швы (лиды, БД-слой, вывод
контента на публичных страницах) ты сводишь сам. JWT здесь stateless без
refresh — для старта норм, для продакшена добавь отзыв сессий. Код проверен
на синтаксис, но не запускался на твоей БД.

Чтобы я сшил CMS с твоим кодом точно — под твою авторизацию, твою схему лидов,
твой стиль роутов — пришли репозиторий, и я допроектирую без догадок.
